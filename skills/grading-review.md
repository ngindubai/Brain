# Skill: grading review

Run monthly, or on request. Keeps focus honest.

## Steps

1. Read [[00-dashboard/priorities]] and every area and project.
2. Re-grade each item against [[reference/priority-system]]: P1, P2, or P3, plus status.
3. Surface the problems:
   - P1 items that are stalled (no real progress since last review).
   - Items eating time but graded P2 or P3.
   - P3 items that should be cut outright.
4. List every open loop across all areas, ranked.
5. Recommend the month's focus: the three or four things to actually push, and what to drop.
6. Update the priorities board and write the review to `00-dashboard/daily/YYYY-MM-monthly-review.md`.

## Rules

- Be blunt. If a P1 has not moved in a month, say so and ask whether it is really P1.
- Keep P1 small. Push back if the board grows more than four P1s.
