## Late items from 1 July (not previously logged)

Four items published on 1 July, after yesterday's report was filed, are worth a short note rather than a full write-up since they're a day old and not developing further today.

**X ships a hosted MCP server.** X now runs its own Model Context Protocol server, letting tools like Claude, Cursor and Grok Build connect to a user's X account without a custom integration. Access is read-only — search, read, look up users, analyse trends and conversations — and explicitly excludes X's Write API, so no autonomous posting through this route. X joins GitHub, Slack, Notion, Stripe and Salesforce in offering an official MCP server, which is a genuine adoption data point for the MCP-ecosystem open loop below. Source: TechCrunch | https://techcrunch.com/2026/06/30/x-now-offers-an-mcp-server-to-make-its-platform-easier-for-ai-tools-to-use/

**LangChain: OpenWiki, an open-source agent for repo documentation.** A CLI/agent that auto-generates and maintains a structured wiki for a code repository, referenced from `CLAUDE.md`-style instruction files rather than dumped wholesale into context, kept current via scheduled GitHub Actions that diff commits. This is directly relevant to how Orwell's own repos could give Claude Code durable, low-token-overhead context instead of bloating instruction files — worth a look next time a repo's CLAUDE.md is getting unwieldy. Source: LangChain Blog | https://www.langchain.com/blog/introducing-openwiki-an-open-source-agent-for-repo-documentation

**LangChain: "How to Use RLMs in Deep Agents."** A follow-on to the Dynamic Subagents/sandboxing/Wiki Memory arc logged yesterday — covers reasoning-language-model patterns inside the Deep Agents framework. Filed as a pointer only; no deep read done today. Source: LangChain Blog | https://www.langchain.com/blog/how-to-use-rlms-in-deep-agents

**Venice AI becomes a unicorn.** $65M Series A led by Dragonfly (crypto-focused VC) at a $1B valuation, Venice AI's first external raise. Two years old, already profitable on ~$70M annualised run-rate revenue, 3M+ active users, 1.7M API calls/day. The pitch: privacy-first routing across 200+ models (open-source "uncensored" models self-hosted, closed-source queries proxied to OpenAI/Anthropic with client-side encryption, nothing stored on Venice's systems). Not Orwell-relevant directly, but a data point that a privacy/no-logging layer in front of frontier models is a fundable wedge on its own. Source: TechCrunch | https://techcrunch.com/2026/07/01/venice-ai-becomes-a-unicorn-with-65m-series-a-as-its-privacy-first-ai-platform-takes-off/
