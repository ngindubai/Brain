## Story 1: Cfx.re Publishes First Technical Detail on FiveM for GTA V Enhanced

**Source:** Cfx.re forum | https://forum.cfx.re/t/development-update-2-fivem-for-gtav-enhanced/5412576
**Related vault notes:** [[Products]]

Cfx.re's "Development Update #2" (posted 1 July, the first real technical detail since yesterday's report only had the placeholder "coming months" line — flagging it now as it's still the top FiveM story with nothing further landing today) lays out several concrete changes for NORTHLANDS to plan around:

- **C# runtime moves from Mono to .NET 10** — a refreshed scripting API with NuGet package support, described as backward compatible with existing resources.
- **VoIP is being fully rebuilt**, replacing Mumble, with the same feature parity but better security; dedicated voice servers remain supported.
- **Connection overhaul**: no more background preload on launch, no forced restarts when switching servers, faster disconnects, and a new WebSocket-based handshake replacing the current UDP-dependent one — framed explicitly as better DDoS protection, relevant for any operator who has dealt with UDP flood attacks.
- **Enhanced ships as a separate client** (via fivem.net), with Legacy staying on the current client — confirming the non-forced, dual-track migration path flagged in prior reports.
- **Asset Escrow support will NOT be in the early access build.** This is the line to flag hardest for NORTHLANDS: any escrowed or paid third-party scripts currently in use will not work on Enhanced at launch of early access.
- Cfx.re says more release-timing detail is coming "later this month."

**NORTHLANDS action:** No urgent change, but this converts last week's abstract migration-timing question into a concrete pre-migration checklist — audit which current scripts are escrow-locked, since those specifically will block an early Enhanced move regardless of when the timing announcement lands.

No new Rockstar Newswire post today; GTA VI terms (November 19 launch, $79/$99 editions, preorders live since 25 June) are unchanged and not re-detailed here.
