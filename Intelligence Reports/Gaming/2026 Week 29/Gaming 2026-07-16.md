## 1. FiveM for GTAV Enhanced: Development Update #3 sets an early access date — 21 July

**Source:** [Cfx Forum, "Development Update #3: FiveM for GTAV Enhanced"](https://forum.cfx.re/t/development-update-3-fivem-for-gtav-enhanced/5415045) (posted 15 July); [RockstarINTEL](https://rockstarintel.com/fivem-for-gta-v-enhanced-gets-development-update/)

This is the update NORTHLANDS has been waiting on. Cfx.re confirmed FiveM for GTAV Enhanced enters early access on **21 July 2026** — the first firm date the project has had since work began. The update covers real technical detail worth planning against ahead of that date:

- **New installer**: separate folder selection for cached server files, desktop shortcuts, built-in uninstall/repair.
- **Networking rebuild**: dropped the third-party sync-frame library in favour of raw UDP packets; sync tick rate jumps from 30-40 to 120 updates per second; multi-core entity culling cuts pop-in on high-population servers; explicit plans to push past the current 2,048-player cap via upcoming community playtesting.
- **Scripting runtimes**: .NET 10 support for C# alongside NodeJS 26 (server) and V8 14.6.202 (client) for JavaScript; state bags get faster sets, lower networking overhead and selective replication.
- **Server component**: renamed, with RAM usage cut up to 50% and better CPU efficiency across platforms.
- **Dev tooling**: developer mode now requires explicit server-side enablement; the legacy profiler is replaced with Perfetto integration; Prometheus metrics expand from 3 to over 80.

Existing Lua/JS/C# scripts remain compatible per the project's earlier March update, though some FiveM code is no longer open source. With five days' notice, this is the first genuinely actionable item this brief has flagged for NORTHLANDS in weeks — see What this means for Orwell.
